# Implementation Plan

- [ ] 1. (P) ログエントリの HTML レンダリングモジュールを実装する
- [x] 1.1 ContentBlock の種別ごとの HTML 変換を実装する
  - テキストブロックをプレーンテキスト表示として変換する
  - ツール呼び出しブロック（ToolUseBlock）を折りたたみ可能な入力パラメータ表示として変換する（`<details>/<summary>` パターン）
  - 思考ログブロック（ThinkingBlock）を既存の thinking セクションと同じ折りたたみ形式として変換する
  - ツール実行結果ブロック（ToolResultBlock）をエラーフラグ付きで変換する
  - すべてのテキストコンテンツに XSS エスケープを適用する
  - _Requirements: 1.3, 1.4_

- [x] 1.2 ロール・タイムスタンプ・コンテンツを含む単一ログエントリの HTML 表示を実装する
  - user/assistant のロール区別をバッジまたはラベルで視覚化する
  - タイムスタンプを各エントリのヘッダーに表示する
  - assistant エントリの ContentBlock リストを 1.1 の変換関数を用いて一覧表示する
  - user エントリのコンテンツ（文字列または ContentBlock リスト）を適切に表示する
  - isMeta フラグが true のエントリを視覚的に区別して表示する
  - _Requirements: 1.2_

- [x] 1.3 エントリリストの表示と 1,000 件超のパフォーマンス対応を実装する
  - エントリを時系列順にスクロール可能なコンテナ内に表示する（CSS max-height + overflow-y: auto）
  - エントリ数が 1,000 件を超える場合、最初の 200 件のみを表示し残りを非表示にする
  - 非表示エントリを一括表示できる「残り N 件を表示」ボタン要素を出力する
  - エントリが空の場合に「ログエントリが見つかりません」メッセージを表示する
  - _Requirements: 1.1, 1.5, 4.5_

- [x] 1.4 サブエージェントセクションの HTML を実装する
  - agent_id をセクションヘッダーに表示し、ページ内アンカー（`id="subagent-{agent_id}"`）を設定する
  - メインセッション詳細ビューへ戻るページ内ナビゲーションリンクを提供する
  - サブエージェントのログエントリを 1.3 のリスト表示関数を用いて一覧表示する
  - _Requirements: 2.2, 2.3, 2.4_

- [x] 1.5 ログ詳細タブ全体を生成するメイン公開関数を実装する
  - メインセッションのログエントリセクションを生成する（1.3 を利用）
  - サブエージェントエントリが存在する場合のみ、各サブエージェントセクションを生成する（1.4 を利用）
  - サブエージェントが存在しない場合、サブエージェントセクションを出力しない
  - `agent_link_map` を受け取り、各エントリのレンダリング時にリンク情報として渡す
  - _Requirements: 2.1, 2.5, 4.1_

- [x] 2. (P) Agent/Task 起動箇所とサブエージェントの位置ベースマッピングを構築する
  - メインセッションのエントリを走査し、Agent および Task ツール呼び出しを登場順に収集する
  - 収集した呼び出しの N 番目を、サブエージェントエントリの N 番目のキー（ファイル名由来、挿入順）に対応付ける
  - 対応するサブエージェントが存在しない呼び出しはマッピングに含めない
  - 結果を `ToolUseBlock.id → subagent_entries キー` の辞書として返す
  - _Requirements: 3.1_

- [ ] 3. HtmlReporter を拡張してログ詳細タブを HTML レポートに統合する
- [x] 3.1 レポーター生成メソッドのインターフェースを ParsedSession とリンクマップを受け取るよう変更する
  - `generate()` の引数に `parsed: ParsedSession` と `agent_link_map` を追加する
  - `_build_html()` に `log_tab_html: str` を引数として追加し、タブパネルに組み込む
  - `SESSION_DATA` に埋め込む `SessionReport` の JSON 形式は変更しない
  - _Requirements: 4.3_

- [x] 3.2 ログ詳細タブの定義とナビゲーションリンクを追加する
  - `_TAB_DEFS` に「ログ詳細」タブを追加する
  - 対応するタブパネルに `render_log_detail_tab()` の出力を組み込む
  - 既存タブとの視覚的な一貫性を維持する
  - _Requirements: 4.1, 4.4_

- [x] 3.3 Agent/Task 起動箇所のハイライトとサブエージェントへのページ内リンクを実装する
  - Agent/Task ToolUseBlock エントリに視覚的なハイライトスタイル（CSS クラス）を適用する
  - `agent_link_map` に対応エントリが存在する場合、サブエージェントセクションへのページ内リンク（`href="#subagent-{agent_id}"`）を表示する
  - `agent_link_map` に対応エントリが存在しない場合、リンクなしで通常表示する
  - 1,000 件超エントリの「残りを表示」ボタンに対応する JS イベントハンドラーを `_JS` に追加する
  - _Requirements: 3.2, 3.3, 3.4, 4.5_

- [x] 4. SessionAnalyzer でリンクマップ構築を呼び出し、HtmlReporter にパイプラインを接続する
  - タスク 2 で実装したマッピング構築処理を `SessionAnalyzer.run()` から呼び出す
  - 変更後の `HtmlReporter.generate()` に `parsed` と `agent_link_map` を渡す
  - 生成された HTML に外部リソース参照が含まれないことを確認する（`file://` 完全動作を維持）
  - _Requirements: 4.2_

- [ ] 5. テストを実装する
- [x] 5.1 (P) LogRenderer の境界条件と各 ContentBlock バリアントを単体テストする
  - エントリが空の場合に「ログエントリが見つかりません」が含まれる HTML が返ることを確認する
  - TextBlock・ToolUseBlock・ThinkingBlock・ToolResultBlock が正しく HTML に変換されることを確認する
  - Agent/Task ToolUseBlock に agent_link_map が存在する場合のリンク HTML を確認する
  - agent_link_map に対応エントリがない場合にリンクなしで表示されることを確認する
  - エントリ数が 1,001 件の場合に「残りを表示」ボタン要素が出力されることを確認する
  - サブエージェントが存在しない場合にサブエージェントセクションが出力されないことを確認する
  - XSS インジェクション文字列（`<script>`・`"`・`&` 等）がエスケープされることを確認する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 2.1, 2.2, 2.3, 2.4, 2.5, 3.3, 3.4, 4.5_

- [x] 5.2 (P) リンクマップ構築ロジックの単体テストを実装する
  - Agent ToolUseBlock が 2 件、サブエージェントキーが 2 件の場合に正しくマッピングされることを確認する
  - Agent 呼び出し数よりサブエージェント数が少ない場合、超過分がマッピングに含まれないことを確認する
  - Agent および Task 以外のツール呼び出しがマッピングに含まれないことを確認する
  - _Requirements: 3.1_

- [x] 5.3 HtmlReporter の統合テストを更新してログ詳細タブの生成を検証する
  - `HtmlReporter.generate()` に ParsedSession を渡した場合、生成 HTML に「ログ詳細」タブが含まれることを確認する
  - サブエージェントセクションのページ内アンカー（`id="subagent-{agent_id}"`）が生成 HTML に存在することを確認する
  - SESSION_DATA が引き続き SessionReport の JSON のみを含むことを確認する
  - _Requirements: 4.1, 4.3, 4.4_
